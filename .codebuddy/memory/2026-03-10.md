# 2026-03-10

## Skills 基于 Primitives 原型重构

### 目标
将所有 skill handler 改为通过 `executePrimitive()` 调用底层原型，无法实现的技能在 `SkillTemplate` 中标记 `disabled`。

### 完成的修改

**1. SkillTemplate 接口扩展** (`frontend/src/data/skills.ts`)
- 新增 `disabled?: boolean` 和 `disabledReason?: string` 可选字段

**2. 标记为 disabled 的技能（4个）：**
- `fix-bug` — 需要完整代码仓库访问与分析能力
- `develop-feature` — 需要完整全栈开发环境与多文件编排
- `optimize-perf` — 需要 Lighthouse 性能分析服务集成
- `image-enhance` — 需要 Real-ESRGAN 超分辨率 API 服务

**3. 重写使用 executePrimitive 的 skill handler（6个）：**
- `generate-todo.ts` — 从 `callDifySkill` 改为 `executePrimitive('structured-output')`
- `meeting-notes.ts` — 从 `callDifySkill` 改为 `executePrimitive('text-to-text')`
- `site-analyze.ts` — 从 `fetchArticles/fetchCrafts + callDifySkill` 改为 `executePrimitive('api-call') + executePrimitive('text-to-text')`
- `task-log.ts` — 从 `fetchArticles/fetchCrafts/fetchWorkflows` 改为 `executePrimitive('api-call')` ×3
- `assign-task.ts` — 从 `callDifySkill + createWorkflow` 改为 `executePrimitive('structured-output') + executePrimitive('workflow-engine')`
- 4个 admin skills (system-monitor/user-manage/data-export/prompt-debug) — 从直接 `fetch` 改为 `executePrimitive('api-call')`

**4. 已正确使用 primitives 的 handler（无需修改，25个）：**
ai-chat, crawl-news, summarize-news, query-dashboard, trend-analysis, send-email, send-notification, generate-article, polish-text, generate-outline, news-to-article, generate-component, update-crafts, layout-design, css-generate, generate-image, generate-chart, image-enhance, review-approve, manage-workflow, run-workflow, quality-check, content-review, regression-test, recruit-cat, team-review, cat-training
