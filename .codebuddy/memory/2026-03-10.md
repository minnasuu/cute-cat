# 2026-03-10

## 画布交互体验优化

优化了 workflow-canvas 的拖拽平移和滚轮缩放体验：

### useCanvasViewport hook 重写
- 核心状态用 `useRef` 存储，通过 `requestAnimationFrame` 节流提交到 React state，避免频繁 setState 造成的卡顿
- 滚轮缩放改为乘法式连续缩放（`zoom * (1 - deltaY * sensitivity)`），替换原来的固定步进，手感更平滑
- 区分触控板 pinch（ctrlKey）和鼠标滚轮，使用不同灵敏度
- 添加拖拽惯性滑动效果（松手后惯性衰减）
- 添加 `will-change: transform` GPU 加速提示
- 修复 `handlePointerDown` 闭包捕获过时 viewport state 的 bug（改用 ref 读取最新值）

### WorkflowCanvas 组件优化
- 背景点阵（dot grid）跟随视口变换：点阵大小和位置随 zoom/pan 同步变化，产生"无限画布"效果
- 添加拖拽距离判定（3px 阈值），防止拖拽后误触发 click 事件
- 使用 `onPointerDownCapture` / `onPointerMoveCapture` 在捕获阶段记录拖拽状态

## 工作流执行前预览确认功能

重构了工作流执行流程，从"点击即执行"改为"先预览确认再执行"：

### 核心逻辑
- 新增 `isPreparing` 状态：标记预览确认阶段
- 新增 `editableStepParams` (Map)：存储每步可编辑参数
- `handleRunWorkflow` 拆分为两阶段：
  1. **prepare**：打开弹窗 → 解析每步 paramDefs → 初始化参数值（优先 step.params.value → defaultValue → 类型默认值）
  2. **confirm**（`handleConfirmRun`）：将用户编辑的参数写回 steps → 开始执行

### 预览态 UI
- 竖向列表展示每个步骤：序号 + 猫猫头像 + 角色名 + 技能名 + action
- 每步下方展示参数编辑表单，支持 7 种输入类型：
  - `text` / `url`：单行输入
  - `textarea`：多行输入
  - `number`：数字输入
  - `select`：下拉选择
  - `toggle`：开关
  - `tags`：标签输入（回车添加，点 × 删除）
- 无参数的步骤显示「无需配置参数」
- 步骤之间用箭头连接
- 底部显示「取消」和「▶ 开始执行」按钮

### 执行态
- 与原有逻辑保持一致，水平管道流水线视图 + 实时进度

## StepConfigPanel 参数管理重写

重写了工作流步骤配置面板的参数区域交互逻辑：

### 修改文件
- **`StepConfigPanel.tsx`**：完全重写参数区域
- **`types.ts`**：`StepParam` 接口新增 `value` 字段

### 核心行为
- 选择技能后，从 `skillPool`（或猫猫 skills）的 `paramDefs` 获取该技能绑定的全部参数定义
- **默认全部展开**为可填写的表单（带输入控件），不再只展示只读信息
- 支持 7 种输入类型（text/url/textarea/number/select/toggle/tags）
- **必填项**不可删除，标红色「必填」标签
- **非必填项**右侧显示 × 删除按钮，删除后从 step.params 移除
- 删除后可通过「添加参数」按钮重新添加回来，下拉菜单只显示该技能绑定的、已被删除的参数
- **不可添加**技能绑定以外的参数
- 参数值通过 `onUpdateStep(stepIndex, 'params', newParams)` 实时同步到父组件

## 简化 upstream 参数来源逻辑

优化了工作流 skill 参数的 upstream（上游输出）取值机制，从"需要指定上游字段名"简化为"只需标记来自上游即可"：

### 核心变更
- **移除 `upstreamField` 字段**：`StepParam` 接口不再有 `upstreamField` 属性
- **移除 `UPSTREAM_FIELD_SUGGESTIONS` 常量**：不再提供上游字段建议列表
- **自动提取主体内容**：upstream 参数执行时按优先级自动查找上游输出中的 `text > summary > notes > content > result > html > body > data` 字段，如果都没有则 JSON 序列化整个输出对象
- **UI 简化**：StepConfigPanel 中 upstream 模式从字段选择器改为只读提示（"执行时将自动使用上一步的输出结果"）

### 修改文件
- `types.ts`：移除 `upstreamField` 和 `UPSTREAM_FIELD_SUGGESTIONS`
- `skills.ts`：send-email 的 body 参数移除 `upstreamField: 'text'`
- `StepConfigPanel.tsx`：`UpstreamFieldSelect` 组件替换为 `UpstreamHint` 只读组件，移除 `handleUpstreamFieldChange`
- `TeamDetailPage/index.tsx`：upstream 解析逻辑改为自动提取主体内容
- `WorkflowPanel.tsx`：同上
- `workflow-executor.js`：后端同步简化
