# 2026-03-10

## 工作流编辑器交互升级：列表表单 → 可视化节点画布

将 `WorkflowEditorPage.tsx` 从「左预览 + 右表单」双栏布局重构为「全屏画布 + 浮动面板」的可视化节点画布模式（类似 n8n / Dify 风格）。

**技术方案**：纯 SVG + CSS Transform 自研轻量画布引擎，未引入第三方 flow 库。

**新增文件**（均在 `frontend/src/pages/TeamDetailPage/workflow-canvas/` 下）：
- `canvas-utils.ts` — 节点尺寸常量、自动布局计算、贝塞尔曲线路径生成、缩放范围控制
- `useCanvasViewport.ts` — 画布平移/缩放状态管理 hook（滚轮缩放以鼠标为中心、中键/背景拖拽平移）
- `WorkflowCanvas.tsx` — 画布主体：圆点网格背景、开始节点、步骤节点、添加节点、连线层
- `StepNode.tsx` — 步骤节点卡片（猫猫头像 + 名称 + 技能 + 参数 badge，支持拖拽移动 + 点击选中）
- `EdgeLines.tsx` — SVG 连线渲染（三次贝塞尔曲线 + 箭头 marker + 透明粗线点击区域）
- `EdgePopover.tsx` — 连线编辑弹窗（点击连线修改 inputFrom 数据来源）
- `StepConfigPanel.tsx` — 右侧滑出配置面板（执行猫猫/技能/行为/参数配置，完整复用原有表单字段）
- `BasicInfoDrawer.tsx` — 基本信息编辑抽屉（名称/描述/触发方式/定时规则/持久化）
- `AiGenerateDialog.tsx` — AI 智能建流弹窗（浮动触发，居中模态）
- `CanvasToolbar.tsx` — 画布左下角毛玻璃工具栏（缩放 + 自动排列 + 基本信息 + AI 建流）
- `SuggestionBanner.tsx` — 建议模式顶部横幅（可展开查看建议猫猫/技能列表）

**修改文件**：
- `WorkflowEditorPage.tsx` — 全面重构：移除左右两栏布局和列表表单，改为画布布局 + 浮动组件，新增 `nodePositions`/`selectedStepIndex`/`activeEdgeIndex` 等画布状态。所有核心逻辑函数（handleSave/handleAiGenerate/updateStep 等）原封不动保留。

**设计决策**：
- 节点坐标不持久化到后端，每次打开自动竖向瀑布布局，WorkflowStep 数据模型完全不变
- TypeScript 编译通过零错误

**删除的文件**：`SuggestionBanner.tsx`

**清理的状态**：`suggestionMode`、`suggestedCats`、`suggestedSkills`、`suggestionSummary`、`dismissSuggestion`

**影响的组件**：`WorkflowEditorPage`、`WorkflowCanvas`、`StepNode`（移除 `isSuggestion` prop 和黄色警告样式）、`StepConfigPanel`（移除 `suggestionMode` prop 和警告提示）

## 工作流参数配置改为 skill 严格绑定 + AI 性格 prompt 自动注入

**产品逻辑调整**：
1. "具体行为"字段改为"步骤描述"（可选 textarea），仅用于说明备注，不影响 AI 执行
2. "用户参数配置"改为只读展示，参数由 skill 的 `paramDefs` 严格绑定，不允许手动增删改
3. 接入 AI 的 skill 执行时自动带入猫猫的 `systemPrompt`（性格 prompt）

**前端改动**：
- `StepConfigPanel.tsx` — 移除 `onAddParam`/`onUpdateParam`/`onRemoveParam` props；"具体行为" input 改为"步骤描述" textarea；参数区改为只读展示（显示 label/type/required/description/placeholder）；新增"自动注入性格 Prompt"提示区（当 skill 是 AI 类技能时显示猫猫的 systemPrompt）
- `WorkflowEditorPage.tsx` — 移除 `addStepParam`/`updateStepParam`/`removeStepParam` 函数和对应 props 传递；`TeamCat` 类型新增 `systemPrompt` 字段

**后端改动**：
- `workflow-executor.js` — `executeStep()` 新增 `catSystemPrompt` 参数；`executeWorkflow()` 在执行每步前查询猫猫的 `systemPrompt` 并注入到 AI 类技能的 system prompt 前面（性格 + 技能 prompt 叠加）
