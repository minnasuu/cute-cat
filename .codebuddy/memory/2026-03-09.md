## 全局 AI 默认模型从 Gemini 切换为 Qwen

将所有 AI 调用的默认模型从 Gemini 改为 Qwen，Gemini 接口代码保留供后期扩展。

修改文件清单：
- `backend/routes/dify.js` — `/models` 和 `/skill` 路由的 fallback 默认值 `|| 'gemini'` → `|| 'qwen'`
- `backend/workflow-executor.js` — `callAI` 函数 fallback 默认值
- `frontend/src/utils/backendClient.ts` — `_currentAIModel` 初始值 + `fetchAIModels` fallback
- `frontend/src/components/WorkflowPanel.tsx` — 模型初始化检查逻辑
- `frontend/src/skills/generate-image.ts` — 默认 model 参数
- `frontend/src/skills/primitives/text-to-image.ts` — 默认 model 参数
- `frontend/src/data/skills.ts` — 所有 `provider: 'Gemini'` → `provider: 'Qwen'`
- `frontend/src/data/cats/*.ts` (alan, huajiao, huangjin, niannian, xiaohu, xue) — 同上
- `.env.example` — `DEFAULT_AI_MODEL=qwen`
- `docker-compose.yml` — 已经是 qwen，无需修改

## 给 default 猫猫（CAT）装备 13 个原型技能

default 猫只装备 13 个原型技能（每个原型一个），不装备其他业务封装技能。

**关键设计**：直接复用 `primitives/` 目录中的 handler，通过 `wrapPrimitive()` 适配器函数包装为 `SkillHandler`，无需创建独立的 `p-*.ts` wrapper 文件。

- `ai-chat` 保留作为 `text-to-text` 原型的入口（有定制逻辑）
- 其余 12 个原型 (`text-to-image`, `structured-output`, `api-call`, `db-query`, `email-send`, `web-push`, `html-render`, `chart-render`, `browser-action`, `file-io`, `workflow-engine`, `js-execute`) 的 skill id 直接使用 primitiveId

修改文件清单：
- `backend/routes/cats.js` — default 猫模板 skills 精简为 13 个原型技能
- `frontend/src/data/skills.ts` — skillPool 原型技能 id 改为 primitiveId；default 技能组 skillIds 精简为 13 个
- `frontend/src/skills/index.ts` — 用 `wrapPrimitive()` 适配器直接引用 `primitives/*.ts`，删除 12 个 `p-*.ts` 文件
