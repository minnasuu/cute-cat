# 2026-03-09 工作日志

## 修复 Docker 部署后数据库不存在导致无法登录的问题
- **问题**：每次重新 docker 部署后，`cute_cat` 数据库可能不存在，导致 Prisma 迁移失败、后端无法启动、用户无法登录
- **根因**：`POSTGRES_DB` 环境变量仅在 PostgreSQL 容器首次初始化数据卷时创建数据库；如果数据卷被删除或在其他环境全新部署，数据库可能不会被创建。`entrypoint.sh` 之前没有检查数据库是否存在就直接执行迁移
- **修改文件**：
  - `backend/Dockerfile`：新增 `RUN apk add --no-cache postgresql-client` 安装 psql/pg_isready 工具
  - `backend/entrypoint.sh`：重写启动脚本，新增三个阶段：1) 解析 DATABASE_URL 提取连接参数 2) 等待 PostgreSQL 就绪（最多30次重试） 3) 检查并自动创建目标数据库 4) 执行 Prisma 迁移（含 baseline fallback）
  - 还补充了第三个迁移 `20260308000000_add_workflow_fields` 的 resolve 逻辑
