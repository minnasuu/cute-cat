# CuCaTopia 产品概念文档

## 核心概念关系（C 端主路径）

```
用户 (User)
 └── 默认工作台 (Team) —— 技术容器，名称「我的工作台」
      ├── 官方猫猫 (Cat) ×17 —— 各绑定一种历史外形模版 + 固定岗位角色
      │    └── 统一 AIGC 执行入口（占位）—— 文本/图像等生成能力后续聚合
      │
      └── 工作流 (Workflow) ×N
           └── 步骤 (Step) ×N
                ├── 执行猫猫（TeamCat.id）
                ├── 能力标识（技术层 skillId，默认 aigc）
                └── 用户配置 (Params)
```

登录后首页为**工作台**：输入框 + 官方工作流卡片 + 历史执行记录 + 猫咪调用次数统计。团队/多猫管理退居「高级设置」，不再作为一级心智模型。

---

## 概念定义

### 团队 (Team)

数据隔离单元；每位用户注册后自动获得一个带标记的默认团队「我的工作台」，用于挂载 17 只官方猫、内置工作流与执行日志。

- 仍支持创建额外团队（受套餐限制），新建团队时会自动补齐 17 只官方猫模版实例
- 前台叙事以工作台为中心，弱化「我的多个团队」

### 猫猫员工 (Cat)

团队中的 AI 工作者。**当前 C 端仅提供官方猫猫**：每只猫有**岗位角色**（如视觉设计师、产品策划等），**不再向用户暴露「技能」心智**；执行侧统一为 **AIGC** 方向，当前版本为**占位实现**（不接真实生成模型），便于后续接入文本/图像等多模态管道。

#### 官方猫猫（唯一来源）

> **官方猫猫 = 固定外形模版 + 岗位角色 + 统一 AIGC 入口（占位）+ 独立名称/人设文案**

- 外形：17 只与 `appearanceTemplates` 按顺序 **1:1 固定对应**
- 能力：存储层保留 `skills: [{ id: 'aigc', ... }]` 以满足数据结构与工作流 `skillId`，产品叙事统称 **AIGC**
- 不再向普通用户提供「自定义猫猫」、自由配色；管理员可在后台编辑文案与 Prompt

#### 自定义猫猫（暂不面向用户）

原定制能力已从主流程隐藏；产品层保留为后续或企业版能力时可再开放。

## 组成要素

### 外形 (Appearance)

- **用户侧**：每只官方猫呈现一种既定品种配色；不在工作台提供「换外形」入口
- **实现侧**：`official-appearance-palette.js` 与 `themes.ts` 中 `appearanceTemplates` 顺序一致

### 性格 (Personality)

人设主要体现在 `systemPrompt` 与语录；不再在社区/工作台提供「性格模版库」作为定制入口。

### AIGC（生成式协作）

- **用户侧**：所有官方猫围绕生成式内容协作；社区/团队页以「✨ AIGC」统一呈现，不展开技能清单
- **实现侧**：工作流步骤 `skillId: 'aigc'` 时，后端执行器返回占位说明（不调用外部模型）；历史工作流若仍含旧 `skillId`，可按旧分支执行直至用户迁移

### 工作流 (Workflow)

多猫按步骤串联；每步绑定**猫猫 + 能力标识（默认 aigc）+ 行为描述（action）+ 参数**。

**当前主路径（工作台内置种子）**：唯一工作流 **网页制作流水线**（`web-page-builder`）——  
1. 产品策划（米卷）：据用户输入产出**网站架构**  
2. 交互设计师（小虎）：产出**交互链路与关键交互说明**  
3. 视觉设计师（墨墨）：产出**页面风格与视觉方向**  
4. 前端工程师（黄金）：产出**页面实现稿（HTML 等）**  

（新用户工作台在无工作流记录时自动种入；存量团队若已有旧工作流，需自行替换或清空后重建。）

## 表格：概念对照

| 概念 | 用户感知 | 说明 |
|------|----------|------|
| **官方猫猫** | 有岗位、会协作的 AI 同事 | 17 模版，外形与角色固定映射 |
| **AIGC** | 生成式产出（文案/视觉等） | 当前占位；后续统一接模型 |
| **工作流** | 多猫流水线 | 步骤默认 `aigc` |
