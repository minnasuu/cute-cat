# CuCaTopia 产品概念文档

## 核心概念关系

```
用户 (User)
 └── 团队 (Team) ×N
      ├── 猫猫员工 (Cat) ×N
      │    ├── 外形 (Appearance)
      │    ├── 性格 (Personality)
      │    └── 技能 (Skill) ×1~N ─── 基于 ──→ 技能原型 (Primitive)
      │
      └── 工作流 (Workflow) ×N
           └── 步骤 (Step) ×N
                ├── 执行猫猫
                ├── 使用技能
                └── 用户配置 (Params)
```

---

## 概念定义

### 团队 (Team)

用户创建的工作单元，是猫猫员工和工作流的容器。

- 一个用户可以创建多个团队（Free 最多 3 个，Pro/Enterprise 不限）
- 一个团队包含 N 名猫猫员工和 N 个工作流

### 猫猫员工 (Cat)

团队中的 AI 工作者，每只猫猫具备特定的外形、性格和技能，承担不同的职责。

猫猫有两种来源：

#### 1. 官方猫猫（内置）

由平台预设的完整角色，开箱即用。

> **官方猫猫 = 外形 + 性格 + 1 个技能组**

官方猫猫的外形、性格、技能组均已绑定，代表一个完整的职位角色（如项目经理、内容编辑、数据分析师等）。

#### 2. 自定义猫猫（用户创建）

用户自行组合创建的猫猫角色。

> **自定义猫猫 = 外形 + 性格 + 1~N 个技能**

用户可以从外形模版、性格模版中选择（或自定义），并自由搭配任意数量的技能，不受技能组限制。

## 组成要素

### 外形 (Appearance)

猫猫的视觉形象，包含配色方案（身体、围裙、眼睛、耳内等颜色）。

- 平台提供多套外形模版（如花椒橘、冰雪白、像素蓝等）
- 用户也可自定义配色

### 性格 (Personality)

猫猫的行为风格和说话方式，影响 AI 的交互语气。

- 每种性格包含：性格特征标签 + 说话风格描述
- 平台提供多套性格模版（如活泼开朗、冷静理性、温柔体贴等）
- 用户也可自定义性格

### 技能原型 (Skill Primitive)

技能原型是系统最底层的能力引擎，**不对用户暴露**。所有用户可见的「技能」都是对某个原型的封装。

> **技能 = 技能原型 + 预设配置（system prompt / API endpoint / 模板 等）**

原型代表一类通用能力，与具体业务场景无关：

| 原型 ID | 名称 | 描述 | 示例技能 |
|---------|------|------|---------|
| `text-to-text` | AI 文生文 | 文本 → LLM → 文本 | 文章生成、内容润色、资讯摘要、会议纪要 |
| `text-to-image` | AI 文生图 | 文本 → 图片生成模型 → 图片 | AI 绘图 |
| `structured-output` | 结构化输出 | 文本 → LLM → JSON | 大纲生成、代办清单、趋势分析、质量检测 |
| `api-call` | 外部 API | 调用 REST / RSS / Webhook | 资讯爬取、图片增强 |
| `db-query` | 数据库查询 | 查询 PostgreSQL | 数据查询、任务日志 |
| `email-send` | 邮件发送 | SMTP 发送 HTML 邮件 | 发送邮件、推送通知 |
| `web-push` | 推送通知 | Web Push API | （预留） |
| `html-render` | HTML 渲染 | 数据 + 模板 → HTML | 排版布局、组件生成、Crafts 更新 |
| `chart-render` | 图表渲染 | JSON → 可视化图表 | 图表生成 |
| `browser-action` | 浏览器操作 | Puppeteer 自动化 | 回归测试 |
| `file-io` | 文件读写 | 服务端文件操作 | （预留） |
| `workflow-engine` | 工作流引擎 | 触发 / 管理工作流 | 执行工作流、工作流管理、审批流程 |

**设计原则：**

1. **用户不感知原型** — 用户在技能池中看到的都是业务化的技能名称（如「文章生成」「内容润色」），不知道底层调用的是同一个 `text-to-text` 原型
2. **原型可复用** — 同一个原型可以被多个技能复用，通过不同的 `primitiveConfig`（预设 prompt、API 地址等）产生不同的行为
3. **新增技能成本低** — 要增加一个新技能，只需选择一个原型 + 编写预设配置，无需重新实现底层能力

### 技能 (Skill)

猫猫能执行的单项能力，是用户可见的最小功能单元。

- 每个技能定义了：名称、描述、输入类型、输出类型、**所基于的原型**、预设配置
- 技能按类别分为：内容创作、数据分析、视觉生成、沟通协作、开发运维、管理调度
- 示例：文章撰写（基于 text-to-text）、数据查询（基于 db-query）、AI 绘图（基于 text-to-image）、邮件发送（基于 email-send）等

### 技能组 (Skill Group)

面向某个特定角色（职位）的技能组合，是对多个技能的打包。

- 一个技能组 = N 个技能的集合
- 技能组代表一个完整角色应具备的所有能力
- 官方猫猫每只绑定 1 个技能组
- 用户创建自定义猫猫时不必使用技能组，可自由选择单个技能

### 角色 (Role)

猫猫在团队中承担的职位。角色是对猫猫职能的概括描述，与技能组对应。

#### 官方角色（7 只猫猫）

| 猫猫 | 角色 | 外形 | 核心职责 | 技能数 |
|------|------|------|---------|--------|
| **花椒** | Project Manager（项目经理） | 狸花加白 | 任务规划与拆解、分配、审批、工作流管理、团队扩编 | 6 |
| **阿蓝** | Content Editor（内容编辑） | 蓝猫 | 文章撰写、内容润色、大纲生成、资讯整理、会议纪要 | 5 |
| **雪** | Data Analyst（数据分析师） | 踏雪黑猫 | 资讯爬取、摘要分析、数据查询、趋势洞察、网站诊断 | 5 |
| **小虎** | Visual Designer（视觉设计师） | 三花猫 | AI 绘图、图表可视化、组件设计、排版布局、图片增强、样式生成 | 6 |
| **发发** | QA Reviewer（质量审核员） | 美短猫 | 内容审核、质量检测、自动化回归测试 | 3 |
| **年年** | Operations Assistant（运营助理） | 橘猫 | 邮件发送、消息推送、任务日志记录 | 3 |
| **黄金** | Engineer（开发工程师） | 金色暹罗猫 | Bug 修复、功能开发、性能优化、Crafts 更新 | 4 |

**角色设计原则：**

1. **通用专业** — 每个角色代表一个通用的自动化工作岗位，用户无需理解技术细节即可组建团队
2. **职责互补** — 7 只猫覆盖「管理→创作→分析→设计→审核→运营→开发」完整链路
3. **最小必要** — 不设冗余角色，每只猫都在多个工作流中承担关键步骤

## 工作流 (Workflow)

将多只猫猫的技能串联起来，按步骤协作完成复杂任务。

### 官方工作流（内置）

基于成熟方法论预设的工作流模版，开箱即用。

- PDCA 循环、OODA 循环、Design Thinking、Scrum、GTD、AIDA 模型、Kaizen 改善、6W 分析法、RACI 矩阵等
- 每个官方工作流预设了步骤顺序、每步的执行猫猫和使用技能

### 自定义工作流（用户创建）

用户根据实际需求自行编排的工作流。

- 自由定义步骤数量和顺序
- 每步指定执行猫猫和使用的技能
- 支持定时调度执行

---

## 概念关系总结

| 概念 | 组成 | 说明 |
|------|------|------|
| **团队** | N 猫猫 + N 工作流 | 用户创建的工作空间 |
| **官方猫猫** | 外形 + 性格 + 1 技能组 | 平台内置的完整角色 |
| **自定义猫猫** | 外形 + 性格 + 1~N 技能 | 用户自由组合创建 |
| **技能组** | N 个技能的集合 | 对应一个角色（职位）的完整能力 |
| **技能** | 原型 + 预设配置 | 用户可见的最小功能单元 |
| **技能原型** | 底层能力引擎 | 用户不可见，被技能封装调用 |
| **工作流** | N 个步骤（猫猫+技能+配置） | 多猫猫协作执行任务 |
